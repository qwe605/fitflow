## FitFlow — 完整产品方案（赛道二：抖音精选-内容重构）

### 一句话定位
把抖音健身视频转化为个性化、可执行的训练计划，让内容从"看过就忘"变成"随时能用"。

---

### 场景与问题洞察

**目标用户**：每周想练 2-3 次、没有私教的上班族（20-35岁）

**核心痛点**：
- 抖音收藏了 30+ 健身视频，打开健身房不知道今天练什么
- 看视频时觉得"学会了"，实际训练时完全想不起动作细节
- 不同博主教的同一个动作要点不一样，不知道该听谁的
- 没有系统规划，今天练胸明天练胸，缺乏科学安排

**数据支撑**（Demo 时可引用）：
- 抖音健身类内容日均播放量超 10 亿次
- 用户平均收藏/点赞后实际执行率不足 5%
- "收藏即学会"是最普遍的内容消费幻觉

---

### 产品核心流程

```
抖音视频链接/口令 → 字幕提取 → AI 结构化解析 → 个性化适配 → 执行看板 → 训练反馈
```

---

### 功能模块设计

**① 视频导入（入口）**

用户粘贴抖音视频分享链接或口令，系统提取视频字幕/文字稿。
- 支持单视频导入
- 支持批量导入（从收藏夹选多个视频，生成组合计划）
- Demo 阶段：预置 5-8 个精选视频作为快速体验入口，同时支持链接输入

**② SOP 生成器（核心 AI 能力）**

AI 解析视频内容，输出结构化训练方案：
- 动作拆解：名称 + 目标肌群 + 组数 + 次数 + 关键要点
- 常见错误警示（"膝盖不要内扣""腰部不要塌陷"）
- 呼吸节奏提示（"下蹲吸气，起身呼气"）
- 预计完成时间与消耗热量估算

**个性化适配**（区别于千人一面的关键）：
- 用户选择体能水平：入门 / 进阶 / 高阶
- AI 自动调整组数、次数、休息时间
- 入门用户额外生成热身和拉伸步骤

**多视频智能整合**：
- 选择多个视频时，AI 自动去重、排序、组合成完整训练日计划
- 同一动作多个博主教学时，AI 对比要点差异，给出综合建议

**③ 执行看板（最高价值感环节）**

把 SOP 变成可交互的训练引导界面：
- 每个动作卡片：展示要点 + 完成按钮
- 内置组间休息倒计时（默认60秒，可调）
- 实时进度条显示完成百分比
- 动作间语音/震动提示切换下一个

**训练中 AI 陪伴**：
- 点击动作卡片上的"?"可追问（"深蹲膝盖疼怎么办"）
- AI 基于当前视频内容上下文给出针对性回答
- 不是通用搜索，而是"这个教练在这个视频里的建议 + AI 补充"

**④ 训练反馈与记录**

- 完成后生成"今日训练卡"（可分享到朋友圈/抖音）
- 记录完成率（完成了 8/10 个动作）
- AI 给出下次建议："你今天完成了 80%，下次可以尝试增加 1 组"
- 训练日历：可视化展示本周/本月训练频次
- 历史记录：随时回顾过去的训练方案

---

### AI 能力深度说明

| 能力层级 | 具体实现 | 价值 |
|---|---|---|
| L1 基础解析 | 视频字幕 → 结构化 JSON | 信息提取 |
| L2 个性化 | 根据用户水平调整参数 | 千人千面 |
| L3 内容整合 | 多视频对比、去重、组合 | 知识聚合 |
| L4 持续陪伴 | 训练中追问、训练后建议 | 长期价值 |

技术实现：
- 调用 Claude API，使用结构化输出（JSON mode）
- Prompt 设计：角色设定为"专业健身教练 + 运动科学背景"
- 输入：视频字幕文本 + 用户体能水平 + 训练目标
- 输出：固定 JSON schema，前端直接渲染

---

### 技术方案

**技术栈**：
- 前端：React / Next.js（Web 应用，移动端优先）
- 后端：Node.js / Python FastAPI
- AI：Claude API（结构化输出 + 对话能力）
- 字幕提取：调用视频平台字幕接口 / Whisper 本地转录
- 数据存储：localStorage（Demo 阶段）/ Supabase（完整版）

**核心数据结构**：
```json
{
  "video_source": "抖音视频链接",
  "user_level": "beginner | intermediate | advanced",
  "workout_plan": {
    "title": "腿部力量训练",
    "estimated_time": "25min",
    "calories": "180kcal",
    "exercises": [
      {
        "name": "深蹲",
        "target_muscle": "股四头肌、臀大肌",
        "sets": 3,
        "reps": 12,
        "rest_seconds": 60,
        "key_points": ["膝盖方向与脚尖一致", "下蹲至大腿平行地面"],
        "common_mistakes": ["膝盖内扣", "重心前移"],
        "breathing": "下蹲吸气，起身呼气"
      }
    ]
  }
}
```

**团队分工（2-3人）**：

| 角色 | 负责内容 |
|---|---|
| 全栈 A | 后端 API + AI 调用（字幕提取 → Claude 结构化输出 → 对话能力） |
| 全栈 B | 前端 UI（视频导入 + SOP 展示 + 执行看板 + 训练记录） |
| 可选第3人 | 内容准备 + 产品演示脚本 + 海报设计 |

**0:20-0:50 视频导入 + AI 生成**
> "我把一个抖音深蹲教学视频的链接粘贴进来——AI 正在解析视频内容——30秒后，一份结构化的训练计划就生成了。"
> （展示 SOP 结果，强调个性化：选择"入门"级别，组数自动减少）

**0:50-1:20 执行看板演示**
> "进入训练模式。每个动作有要点提示，完成一个打个勾，组间自动倒计时。如果我对某个动作有疑问——点这里问 AI——它会基于这个视频的内容给我针对性回答。"

**1:20-1:40 训练完成 + 反馈**
> "训练结束，生成今日训练卡——这个可以直接发朋友圈。AI 还会告诉我下次可以怎么调整。"

**1:40-2:00 延展愿景**
> "未来，FitFlow 可以接入抖音收藏夹，一键把你收藏的所有健身视频变成一周训练计划。同样的能力也可以扩展到烹饪、装修、学习——任何'看了但没用上'的内容。"

### 与抖音生态的结合点（扣题关键）

1. **输入端**：内容来源是抖音视频（链接/口令导入）
2. **价值端**：把抖音"刷到即忘"的内容变成可执行能力
3. **输出端**：训练卡可分享回抖音/朋友圈，形成内容消费→行动→分享的闭环
4. **延展端**：未来可作为抖音内置工具，在视频播放页直接生成训练计划
